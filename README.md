## Как запустить

### 1. Установка зависимостей

```bash
make bin-deps
```


### 2. Генерация кода из proto файлов

```bash
make generate
```


### 3. Запуск инфраструктуры

```bash
make start-infra
```

Эта команда:
- Запускает PostgreSQL в Docker контейнере
- Применяет миграции базы данных

### 4. Запуск приложения

```bash
make run
```

Приложение запустится на порту `50051` (по умолчанию).

## Полная инструкция по запуску

```bash
# 1. Установить бинарные зависимости
make bin-deps

# 2. Сгенерировать код из proto файлов
make generate

# 3. Запустить PostgreSQL и применить миграции
make start-infra

# 4. Запустить приложение (в отдельном терминале)
make run
```





## Разработка

### Установка зависимостей

```bash
make bin-deps
```

### Создание новой миграции

```bash
make migration
# Введите имя миграции при запросе
```

### Регенерация proto кода

После изменения proto файлов:

```bash
make generate
```

### Проверка proto файлов

```bash
make lint
```
### Проверка гошным линтером

```bash
make go-lint
```

## Остановка приложения

Для остановки приложения нажмите `Ctrl+C` в терминале, где запущен сервер.

Для остановки инфраструктуры:

```bash
make stop-infra
```

Для полной очистки (включая данные БД):

```bash
make clear-infra
```
